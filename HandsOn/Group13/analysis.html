Analysis, license & RNS.

1. Dataset analysis

Dataset: Accidentalidad 2025 (Traffic accidents, City of Madrid) — Madrid Open Data portal.

Schema: 

expID,date,loc1,loc2,numLoc,district,distCode,accidentType,weather,vehicleType,personType,ageRange,sex,utmXcoord,utmYcoord,alcoholPositive,person_id

Notes:

  • The dataset includes information on traffic accidents in Madrid, with fields such as accident case number, date and time, location (including UTM coordinates), district, accident type, vehicle and person type, demographics, and toxicology results. All values are stored as text.
  • The dataset was cleaned and restructured. Unnecessary columns like positiva_droga and lesividad were removed, and rows with blank accident types were deleted. Data was standardized in the estado_meteorológico and localizacion columns, with the latter also being split into separate columns.

3. Resource Naming Strategy

Definition of the strategy for naming resources in the ontology and the data generated from the accident CSV.

  • URI Format

    • Use dereferenceable URIs that allow access to information about the resource. The URIs follow a consistent and human-readable structure:

    http://example.org/{resource_type}/{identifier}
  • URI Domain

    • Use institutional or trusted domains such as datos.madrid.es for official accident data, and example.org for additional entities (districts, streets, intersections, vehicles, persons).

  • Path Structure

    • Organize resources by type and include meaningful identifiers to ensure clarity and persistence:

    http://example.org/vehiculo/{expID}-{numero}
  • URI Patterns
    • Resource Type URI Pattern Example
    Accident http://datos.madrid.es/accidentes/{expID} http://datos.madrid.es/accidentes/2025001234
    District http://example.org/distrito/{district} http://example.org/distrito/centro
    Street http://example.org/calle/{loc1} http://example.org/calle/alcala
    Intersection http://example.org/cruce/{loc1}-{loc2} http://example.org/cruce/alcala-granvia
    Vehicle http://example.org/vehiculo/{expID}-{person_id} http://example.org/vehiculo/2025001234-1
    Person http://example.org/persona/{expID}-{personType}-{person_id} http://example.org/persona/2025001234-conductor-1
